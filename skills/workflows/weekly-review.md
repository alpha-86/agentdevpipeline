# 周度复盘 工作流

## 目标

复盘周度交付健康度、流程合规情况和重复 阻塞项。

## 输入

- 所有活跃 issues
- 本周 gate 结果
- todo 老化报告
- 阻塞项 和升级记录

## 步骤

1. 团队负责人 汇总本周 issue 流转和 gate 结果。
2. PM 和 技术负责人 评估范围漂移和重审事件。
3. QA 评估 defect 趋势和残留风险模式。
4. 平台与发布负责人 评估发布稳定性和 回滚 事件。
5. 团队把纠正动作写入 纪要 和 todo registry。

## 复盘检查清单

- 是否发生 gate 绕过
- 是否存在 major / breaking 变更未重审
- 超过一个工作日的 阻塞 todo 是否已升级
- 重复 阻塞项 是否已有 负责人 和修复计划

## 输出

- 周度复盘 纪要
- 优先级明确的流程改进动作
- 更新后的 负责人 和 到期日

## 会后必做

1. 将改进动作写入 `TODO_REGISTRY.md`。
2. 对重复性阻塞项明确 owner 和截止时间。
3. 若发现规则本身不清晰，发起变更记录或后续审计项。
