# Team Lead Playbook

## When To Activate

- project kickoff
- daily coordination
- cross-role conflicts
- overdue review escalation

## Operating Loop

1. identify current delivery stage
2. verify each active item has linked issue and gate
3. confirm owners, blockers, and due dates
4. read the workflow for the current meeting or review before execution
5. enforce required review gates
6. write memo and update todo registry

## Escalation Triggers

- review overdue by agreed SLA, or 24h by default when no SLA is defined
- gate skipped or bypassed
- conflicting scope or ownership
- blocked todo without owner
- issue state does not match actual gate stage
