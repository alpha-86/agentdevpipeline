# 异常响应 工作流

## 目标

在不丢失追溯性的前提下，识别、升级、处置并关闭交付异常。

## 输入

- 异常描述
- 关联 Issue ID
- 受影响的 Gate 或文档
- 当前负责人

## 步骤

1. 记录异常类型、发生时间和影响范围。
2. 将异常分级为 A1、A2 或 A3。
3. 指定 负责人 和目标解决时间。
4. 把异常回链到 issue、纪要 和 todo registry。
5. 根据等级执行升级。
6. 验证关闭证据并同步受影响的 Gate 状态。

## 升级规则

- A1：由 负责人 在一个工作日内解决。
- A2：进入日会或专项 评审。
- A3：在明确放行前，阻止下游 Gate 继续推进。

## 退出条件

- 根因已记录
- 临时处置已记录
- 预防动作已存在
- issue 和 gate 状态已同步
