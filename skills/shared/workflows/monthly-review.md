# 月度复盘 工作流

## 目标

评估流程质量趋势，并决定下个月的结构性改进项。

## 输入

- 当月 weekly 评审 memos
- gate pass/fail 趋势摘要
- change record 摘要（minor/major/breaking）
- release incidents 和 回滚 摘要

## 步骤

1. 团队负责人 汇总月度流程与合规指标。
2. PM 评估需求质量和验收变动。
3. 技术负责人 评估架构债务和返工成本。
4. QA 评估质量趋势和测试缺口热点。
5. 平台与发布负责人 评估发布可靠性和运行风险。
6. 团队定义下月流程改进项和 负责人。

## 复盘检查清单

- 流程规则是否仍清晰且可执行
- gate 签字质量是否一致
- 需要重审的变更是否都被重审
- 重复风险是否已有预防控制

## 输出

- 月度流程复盘 纪要
- 已批准的改进 backlog
- 必要时更新标准或模板
