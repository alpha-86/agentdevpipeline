# 评审严谨性与上下文恢复机制

## 目标

在会话中断、上下文清空、跨人接手等场景下，保持流程连续性与评审质量不下降。

## 评审严谨性（强制）

- 先读全量必读文档，再给方案结论。
- 先给证据与引用，再给通过/退回判断。
- 对“关键规则”逐项核验，不得凭经验跳过。
- 不允许“假装读完”或“先做再补审查”。

## 必读材料顺序

1. 当前 Issue 及其最新状态
2. 对应阶段文档（PRD/Tech/QA/Release）
3. 相关 memo 与 todo
4. 角色约束与 workflow 规则
5. 最近变更记录（change record）

## 上下文恢复触发条件

- 对话历史被清空
- agent 重启或切换执行环境
- 任务由其他角色接手
- 连续超过一个工作日未更新

## 恢复流程

1. 从 Issue 恢复当前阶段与未完成动作。
2. 读取最新 PRD/Tech/QA/Release 文档状态。
3. 读取最近三条评审结论与变更记录。
4. 输出“恢复摘要”：当前阶段、阻塞、下一步、风险。
5. 再进入执行，不得跳过恢复摘要。

## 恢复摘要模板

- 当前 Issue：
- 当前阶段：
- 最近一次 Gate 结论：
- 未完成 Todo：
- 阻塞与风险：
- 下一步动作：
- 需要谁决策：

## 交接规则

- 交接方必须提供“阶段快照 + 待办 + 风险”三件套。
- 接手方必须复核并确认“输入完整性”后再执行。
- 输入缺失时必须先补文档，不得继续下游动作。
