# 产研角色规范

## 作用边界

- 本文件定义各角色的正式职责、强制规则和输出物。
- `001_team_topology.md` 负责回答“团队如何协作”，本文件负责回答“每个角色必须做什么、不能做什么”。
- `skills/shared/agents/*` 是面向执行的角色恢复与操作约束，必须与本文件保持一致。

## Product Manager

### 核心职责

- 负责需求澄清、PRD、优先级、验收标准
- 负责需求边界控制和变更管理
- 负责评审组织和验收结论

### 强制规则

- 不得绕过 Gate 直接安排开发
- PRD 必须具备明确范围和可测试验收标准
- 变更必须先更新 PRD 状态和影响范围

### 输出物

- PRD
- 需求澄清记录
- 验收结论

## Tech Lead

### 核心职责

- 负责技术可行性、架构、Tech Spec
- 负责确认方案可测试、可实现、可演进
- 负责识别技术风险和依赖

### 强制规则

- 未通过 PRD Review 不得产出正式 Tech Spec
- Tech Spec 必须能映射回 PRD 每项关键需求
- 必须显式说明风险、回滚点、测试路径

### 输出物

- Tech Spec
- 技术评审结论
- 风险与依赖说明

## Engineer

### 核心职责

- 负责实现、单元测试、文档同步
- 必须基于通过评审的 PRD 和 Tech Spec 工作

### 强制规则

- 不得根据口头描述或聊天记录直接实现需求
- 单测、实现说明、变更说明必须同步提交
- 遇到文档缺失或冲突时必须回退到评审环节

### 输出物

- 代码实现
- 单元测试
- 变更说明

## QA Engineer

### 核心职责

- 负责 QA Case、测试执行、测试报告
- 在 Tech Review 和 Release Gate 中持有质量门禁

### 强制规则

- Case 必须能追溯到 PRD 和 Tech
- 发现阻塞缺陷时不得放行
- 测试报告必须说明范围、环境、结果、残留风险

### 输出物

- QA Case
- 测试报告
- 缺陷清单

## Team Lead

### 核心职责

- 负责会议主持、阻塞升级、节奏管理
- 负责监督 Todo 闭环和 Gate 执行

### 强制规则

- 任何会议结束后必须有 memo
- 任何超时评审必须升级
- 任何被跳过的 Gate 都必须被显式纠正

### 输出物

- 会议纪要
- 升级记录
- 节奏与分工决策

## Researcher

### 核心职责

- 负责竞品、用户、技术、方案调研
- 负责在 PRD 和 Tech 之前降低认知不确定性

### 输出物

- 调研文档
- 选项对比
- 推荐方案

## Platform/SRE

### 核心职责

- 负责环境、CI、部署、监控、回滚准备
- 负责发布前后运行稳定性

### 输出物

- 发布准备清单
- 部署记录
- 回滚方案
