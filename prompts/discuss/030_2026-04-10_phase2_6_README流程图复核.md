# 030 - 2026-04-10 Phase 2.6 README 流程图复核

## 目标

确认根 `README.md` 中给用户看的完整流程图，是否准确体现了从 `hedge-ai` 迁出的主线，而不是停留在抽象概括。

## 当前项目原有问题

旧版 README 的流程图虽然能表达“大致有两个 Human Review 节点”，但还不够准确，主要缺：

- 没把 `Issue First` 放在流程开头写清
- 没写出 “PM 先回到问题本身”
- 没写出文档 PR 与代码 PR 的双阶段形态
- 没写出 `Issue Comment Gate`
- 没写出 Issue 最终由 Human 关闭
- 没写出 PMO 是按 PR 触发的主动检查，而不是另起一条主流程

## 本次落地调整

### 1. 重写 README 的完整流程图

现在 README 中的主线已经明确成：

1. 创建主 Issue
2. PM 澄清问题并持续 comment
3. PRD
4. Tech Review + QA Gate
5. QA Case Design
6. 文档 PR
7. Human Review #1
8. 开发与测试
9. 代码 PR
10. Human Review #2
11. Issue Comment Gate
12. Release / 验收 / Human 关闭 Issue
13. PMO 按 PR 主动检查，日汇总 / 周复盘 / 月复盘

### 2. 同步收紧 README 的核心原则摘要

README 顶部的原则摘要也已同步到当前真实主线：

- Issue First
- 问题先于方案
- Human Review
- 双阶段交付
- 正式留痕
- Gate 不可绕过
- 独立治理职责
- 迁移边界

### 3. 修正 README 中的剩余表述

“现在还缺什么” 里不再继续写 `Phase 1` 已完成的“逐文档覆盖检查”，改成当前真实缺口：

- 插件入口文档
- 已迁移资产继续向插件入口收紧
- 依赖与复用策略收口

## 结论

README 现在更接近用户真正需要看到的内容了：

- 它说明了这个项目解决什么问题
- 也把主流程画清楚了
- 并且没有再把流程图画成脱离源文档的泛化版本

下一步将进入 `Phase 3`，开始把 README、插件入口、plugin 元数据、CHANGELOG 收成真正的插件资产。
