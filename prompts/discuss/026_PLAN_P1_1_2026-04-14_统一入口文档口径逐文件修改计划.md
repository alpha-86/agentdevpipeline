## 三、统一口径基线

本阶段统一后的文档基线应为：

### 项目定位

- AgentDevFlow 是独立的通用产研多 Agent 研发交付流程资产包
- 重点是 Issue 主线、文件交付物、双阶段 PR、Gate 门控、人机协同与多 Agent 协作

### 正式角色集

- `team-lead`
- `product-manager`
- `architect`
- `qa-engineer`
- `engineer`
- `platform-sre`
- `pmo`

### 核心主线

- GitHub Issue 是正式主线
- 文件交付物是正式交接物
- 双阶段 PR 引入 Human Review
- QA / Engineer 形成交付质量闭环
- PMO 形成流程改进闭环
- Telegram 是移动端实时沟通入口，不是正式交付主线
