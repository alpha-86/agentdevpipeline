  - QA Test Report Gate 5：`156-159`

**影响**：

- 但 CI 不会因为签字缺失而失败

**要求修正**：
- 缺签字必须 `exit 1`
- 不允许只打印 warning 后继续通过

### P0-2. CI 检查口径与新的 Gate 文档不一致，自动化规则已分叉

**问题文件**：
- `.github/workflows/doc-pr-checks.yml`
- `prompts/004_delivery_gates.md`

**证据**：
- Gate 文档已改成：
  - PRD Gate 1：`PM + 架构师 + 质量评审人`
  - Tech Gate 2：`PM + 技术评审人 + QA`
  - QA Case Design Gate 3：`PM + 架构师 + Engineer`
  - QA Test Report Gate 5：`QA + PM + Engineer`
- 但 workflow 仍在找：
  - `CTO`、`CRO`、`CSO/CRO`
- 位置：
  - `.github/workflows/doc-pr-checks.yml:97-100`
  - `.github/workflows/doc-pr-checks.yml:117-120`
  - `.github/workflows/doc-pr-checks.yml:182-183`

**影响**：
- 文档规则与 CI 检查规则不一致
- 即使文档按新规则填写，CI 也可能误报缺失
- 这会直接破坏 Gate 的可信度

**要求修正**：
- workflow 中的角色检查必须与 `prompts/004_delivery_gates.md` 完全一致
- 成功输出里的 Gate 说明也必须同步
