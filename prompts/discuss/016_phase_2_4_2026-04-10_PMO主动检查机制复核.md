# 028 - 2026-04-10 Phase 2.4 PMO 主动检查机制复核

## 对照源文件

- `hedge-ai/prompts/V3.0/change_record/V2.6_PMO-Agent引入_流程合规主动检查机制.md`

## 本次只复核什么

本次只复核这四点：

1. PMO / Process Auditor 是独立角色
2. 每次 PR 都是独立触发点
3. PMO 负责记录和升级问题，不替代正式评审角色
4. 问题按天汇总，并对 `P0 / P1` 立即升级

## 当前项目原有问题

当前仓库虽然已经有：

- `prompts/zh-cn/014_process_compliance_and_audit.md`
- `skills/shared/agents/process-auditor.md`

但还不够接近 `hedge-ai V2.6` 的原始边界，主要缺：

- 没明确写出“PMO 是独立角色，但不替代其他评审角色”
- 没明确写出“每次 PR 都是独立触发点”
- 没明确写出“默认不直接阻塞流程，只记录和推动纠正”
- 没有明确的 `docs/pmo/issues/YYYY-MM-DD.md` 留痕落点

## 本次落地调整

### 1. 补齐角色边界

现在已经明确：

- Process Auditor 是独立治理角色
- 不能替代 PM、Tech Lead、QA、Platform/SRE 的正式签字职责
- 不得把自己当作新的强制签字 Gate

### 2. 补齐 PR 触发机制

现在已经明确：

- 每次 PR 都是一次独立审计触发点
- 文档 PR 检查：PRD、Tech、QA Case Design、评审记录
- 代码 PR 检查：代码、测试报告、验收记录、文档一致性

### 3. 补齐问题处理边界

现在已经明确：

- Process Auditor 默认不直接阻塞流程执行
- 主要职责是发现、记录、推动纠正和升级严重问题
- `P0 / P1` 问题必须立即通知 Team Lead
- 不能自行关闭问题

### 4. 补齐记录落点

新增：

- `docs/pmo/issues/README.md`

用于约束：

- 每天一个文件
- 当天问题持续追加
- 问题字段和关闭责任归属

## 结论

`PMO / Process Auditor` 这条线，现在比之前更接近 `hedge-ai V2.6` 的原始定义了，尤其是：

- 从“泛审计角色”收紧到“独立治理角色”
- 从“定期巡检”收紧到“每个 PR 主动触发”
- 从“发现问题”收紧到“记录、升级、按天汇总”

后续仍需继续推进的部分：

- 插件入口层如何承载 PMO 触发说明
- README 流程图如何体现 PMO 不替代签字、只做主动检查
