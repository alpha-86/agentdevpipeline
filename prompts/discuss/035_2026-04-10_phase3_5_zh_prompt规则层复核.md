# 035 - 2026-04-10 Phase 3.5 zh prompt 规则层复核

## 目标

确认 `prompts/zh-cn/` 是否已经足以构成插件规则层，而不是仅仅作为中文文档集合存在。

## 本次复核范围

重点复核这些主干规则：

- `001_team_topology.md`
- `003_document_contracts.md`
- `004_delivery_gates.md`
- `005_meeting_and_todo.md`
- `006_agent_creation_contract.md`
- `007_issue_driven_orchestration.md`
- `013_github_issue_and_review_comments.md`
- `014_process_compliance_and_audit.md`

- `019_dual_stage_pr_and_three_layer_safeguard.md`
- `020_issue_comment_gate_and_artifact_linkage.md`
- `021_platform_checks_and_gate_automation.md`

## 复核结论

### 1. 已基本形成插件规则层

当前 `prompts/zh-cn/` 已经覆盖插件主干规则所需的关键内容：

- 团队结构
- 文档契约
- Gate 规则
- 会议与 Todo 机制
- Agent 创建契约
- Issue First
- Comment Gate
- PMO 主动检查
- Human Review
- 双阶段 PR
- 平台最小检查

这意味着它已经不再只是“中文说明目录”，而是插件规则层本身。

### 2. 当前最关键的缺口

当前更真实的缺口仍然是三类：

1. `github-issue/SKILL.md` 的插件入口映射还不够直接  
   现在规则内容已经在 `007`、`013`、`020` 中，但还缺更清晰的插件入口承载。

2. `review-org/SKILL.md` 的插件入口映射还不够直接  
   现在更多体现在 `009_review_rigor_and_context_recovery.md` 与 `prompts/discuss/` 审计口径中。

3. `015_review_evaluation_dimensions.md` 还是维度说明  
   它还没有形成更接近 `hedge-ai` evaluation checker 的独立规则承载层。

在 `prompts/zh-cn/README.md` 中补充了正式判定：

## 结论口径

因此，这一项的结论是：

- `prompts/zh-cn/` 已经能够作为插件规则层成立
- 但仍需继续把 `github-issue`、`review-org`、evaluation checker 的插件承载方式收紧
