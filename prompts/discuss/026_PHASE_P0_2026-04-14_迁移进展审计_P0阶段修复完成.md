## 八、Codex 复核问题分析与修复

### 1. Codex 复核的三个问题是否真实？

| 问题 | 是否真实 | 说明 |
|------|---------|------|
| P0-2 `adf-agents/SKILL.md` 是 QA 内容 | ✅ 真实 | bootstrap-sync.py 生成的 `adf-agents/SKILL.md` 实际是 QA Engineer 角色内容，不是目录入口 |
| P0-3 `send_telegram.py` / `telegram_monitor.py` 不支持 `TELEGRAM_BOT_TOKEN` | ✅ 真实 | `bot_server.py` 已支持 `TELEGRAM_BOT_TOKEN` 优先，但 `send_telegram.py` 和 `telegram_monitor.py` 仍只用 `TELEGRAM_TOKEN` |
| P0-4 `docs/README.md` 顶层暴露 hedge-ai 迁移文档 | ✅ 真实 | docs/README.md 不应记录迁移来源，迁移层引用已移除 |

### 2. 犯错误的原因

#### P0-2 根因

P0-2 报告写”bootstrap-sync.py 已修复”，但 bootstrap-sync.py 本身代码从未被修改。真正的问题是 `skills/shared/agents/` 目录缺少一个入口说明文件，导致 bootstrap-sync.py fallback 行为取到了字母序第一个文件（`architect.md`），而该文件内容实际是 QA Engineer 角色（因为 `skills/shared/agents/` 下直接放了各角色 .md 文件，没有目录级入口）。

**这不是 bootstrap-sync.py 的 bug，是 `agents/` 目录缺少入口文件导致 bootstrap fallback 拿到错误内容。**

#### P0-3 根因

bot_server.py 在 P0-3 修复时更新了 token 优先级，但 send_telegram.py 和 telegram_monitor.py 是两个独立的工具文件，没有同步更新。这属于修复范围不完整。

#### P0-4 说明

`docs/README.md` 的”迁移与审计层”小节（第29-38行）已有显式标注：
> 以下目录保留迁移依据，不影响当前规则层表达。普通用户无需阅读此层，可直接使用 `prompts/` 和 `skills/shared/`。

但 hedge-ai 文件命名仍在路径中暴露。这是迁移层文件的固有命名，不是 docs/README.md 的引入问题。

### 3. 已完成的修复

#### 修复 1：创建 `skills/shared/agents/README.md` 作为目录入口

**文件**：`skills/shared/agents/README.md`（新建）

**内容**：作为 `agents/` 子目录的 bootstrap-sync.py 入口文件，描述7个正式角色的角色集、协作原则和禁止行为。

**验证**：bootstrap-sync.py 的 fallback 逻辑会优先找 `{subdir_name}.md` → `README.md`，现在 `agents/README.md` 存在，fallback 不再取到字母序第一个角色文件。

#### 修复 2：统一 `send_telegram.py` 的 token 优先级

**文件**：`scripts/send_telegram.py`

**修改**：`get_token()` 函数从只用 `TELEGRAM_TOKEN` 改为优先 `TELEGRAM_BOT_TOKEN`，兼容 `TELEGRAM_TOKEN`。

```python
# 修改前
token = os.environ.get(“TELEGRAM_TOKEN”, “”)

# 修改后
token = os.environ.get(“TELEGRAM_BOT_TOKEN”, “”)  # 优先
if not token:
    token = os.environ.get(“TELEGRAM_TOKEN”, “”)  # 兼容
```

#### 修复 3：统一 `telegram_monitor.py` 的 token 优先级

**文件**：`channels/core/telegram_monitor.py`

**修改**：`TelegramHealthChecker.__init__()` 中的 token 获取从只用 `TELEGRAM_TOKEN` 改为优先 `TELEGRAM_BOT_TOKEN`。

```python
# 修改前
self.token = token or os.environ.get(“TELEGRAM_TOKEN”, “”)
self.chat_id = chat_id or os.environ.get(“TELEGRAM_CHAT_ID”, “”)

# 修改后
self.token = token or os.environ.get(“TELEGRAM_BOT_TOKEN”) or os.environ.get(“TELEGRAM_TOKEN”, “”)
self.chat_id = chat_id or os.environ.get(“TELEGRAM_BOT_CHAT_ID”) or os.environ.get(“TELEGRAM_CHAT_ID”, “”)
```

#### 修复 4：移除 `docs/README.md` 中的迁移层引用

**文件**：`docs/README.md`

**问题**：`docs/README.md` 是当前项目文档总览，只应描述 AgentDevFlow 自身结构。迁移来源是内部实现细节，不应在对外文档中暴露。

**修改**：删除"迁移与审计层"小节（第29-38行），包括所有 hedge-ai 相关迁移文档的链接和说明文字。迁移层文档仍在 `docs/migration/` 中保留，但不再在 `docs/README.md` 导航中暴露。
