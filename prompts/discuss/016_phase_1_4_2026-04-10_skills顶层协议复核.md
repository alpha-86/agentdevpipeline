# 020 Phase 1.4 skills 顶层协议复核

## 复核范围

- `.claude/skills/TEAM_SETUP.md`
- `.claude/skills/SKILL_PROTOCOL.md`
- `.claude/skills/EVENT_BUS.md`
- `.claude/skills/ANOMALY_LOOP.md`

## 复核结论

### 1. `TEAM_SETUP.md`

可迁移为插件共享入口的部分：

- 团队初始化前置条件
- 角色创建顺序
- 必读文档加载顺序
- 启动后立即进入日会 / kickoff 的约束
- 会话压缩后的恢复入口

不迁移的部分：

- `hedge-ai` 专属 Team 名称
- 业务角色集合
- 业务领域标签配置细节

### 2. `SKILL_PROTOCOL.md`

可迁移为插件共享协议的部分：

- 事件驱动调用
- Skill / workflow 状态机
- 标准输出格式
- 失败重试与错误分类
- 状态持久化与审计日志思路

结论：

这是插件共享协议主干，不应被视为外围文档。

### 3. `EVENT_BUS.md`

可迁移为插件共享编排能力的部分：

- 事件总线架构
- 事件格式
- 订阅与触发规则
- 状态存储与分发关系

注意：

- hedge-ai 源文档里的事件类型大量带有方案 / 指标 / 验证语义，迁移时只能保留抽象结构，不能保留业务字段。

### 4. `ANOMALY_LOOP.md`

可迁移的部分：

- 异常发现 → 分类 → 改进 → 验证闭环
- 异常优先级
- case 状态机
- 闭环验证

不直接迁移的部分：

- 面向业务文档与业务数据的异常分类口径

## 对插件目标的影响

这四份文档决定了插件是否只是“文档包”，还是有共享启动协议、共享调用协议、共享编排协议、共享异常闭环协议。
