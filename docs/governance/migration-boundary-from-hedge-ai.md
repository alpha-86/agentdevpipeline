# 与 hedge-ai 的迁移边界（强约束）

## 目的

本文件用于明确 `agentdevflow` 与 `hedge-ai` 的关系，防止把业务域特定内容误引入本项目。

## 关系定义

- `agentdevflow` 是独立的通用产研流程编排项目，不是 hedge-ai 的子模块。
- `hedge-ai` 仅作为“流程完备性参考样本”，用于抽取可复用的研发方法学。
- 迁移时只允许抽象“流程机制”，禁止复制“特定业务语义”。

## 允许迁移（可复用）

- 角色协作边界定义方法（职责、输入、输出、签字责任）
- Gate 驱动交付机制（PRD/Tech/QA/Release）
- 文档状态机与评审留痕机制
- Issue 驱动到交付闭环机制
- 研发日会机制（通用研发同步，不含特定业务时段语义）
- Todo 闭环与会议纪要规范
- Agent 创建、编排、恢复上下文、异常处理等工程化机制

## 严禁迁移（禁止引入）

- 特定业务组织角色语义
- 前置/后置阶段等特定业务时段编排
- DBR、异常日报（业务风控语境）等特定业务例会模板
- 指标研究、验证评审、方案辩论（业务域语境）等专用流程
- 风险指标、偏差指标、事务指标、风控阈值等业务域指标和字段

## 研发日会保留规则

- 保留“研发日会”机制：同步进展、阻塞、风险、当日计划、跨角色协同事项。
- 日会输出必须面向通用研发交付，不得包含特定业务口径字段。

## 术语迁移映射（必须改写）

以下内容只允许迁移“机制”，必须替换为通用产研语义：

- “策略评审” -> “方案评审 / 技术方案评审”
- “研究员” -> “Researcher（通用研究角色）”
- “风控检查” -> “质量与风险检查（工程交付语境）”
- “日报” -> “研发进展纪要 / 交付纪要”
- “异常日报” -> “阻塞与风险跟踪纪要”
- “DBR 例会” -> “研发例会 / 日会（工程交付语境）”

若术语无法改写为通用研发语义，说明该内容属于业务域知识，不得迁移。

## 禁用词清单（文档评审）

以下词汇出现在本仓库流程文档中，默认判为越界，需退回修改：

- 前置阶段、后置阶段、启动、结束
- 特定业务角色、规模、库存、风险指标
- 验证、指标、信号、收益、最大偏差（业务域语义）
- DBR（用于业务运营语境时）

说明：当某词在通用工程语义中有歧义时，必须在文档中明确“非业务域语境”。

## 迁移执行检查清单

每次从 hedge-ai 迁移内容时，必须逐条检查：

1. 是否可被表述为“任何软件研发团队都适用”的流程机制。
2. 是否引入了特定业务角色、指标、时段、报表术语。
3. 是否已在文档中替换为通用产研语义。
4. 是否保留了可追溯性（输入、输出、负责人、Gate 条件）。
5. 是否已在当前仓库完成中文落地，并保持单线结构一致。

若第 2 条为“是”，该内容不得进入本仓库。

## 合并门禁（中文主版本）

- 任何来自 hedge-ai 的迁移改动，在中文文档评审时必须显式引用本文件。
- PR 描述中必须包含“可迁移项 / 删除项 / 语义替换项”三段说明。
- 若评审者发现特定业务语义残留，必须阻断合并，不允许“后续再改”。

## Phase 7 边界复核结论

- 根 `README.md` 与 `plugins/agentdevflow/README.md` 当前未发现新的特定业务语义混入主干。
- `skills/shared/agents/*` 与 `skills/shared/workflows/*` 当前未发现新的特定业务角色、特定业务节奏或业务域指标字段残留。
- 当前仓库中出现的”业务域验证/指标”等词，主要用于迁移边界、源清单和禁止迁移说明，不用于插件主干执行语义。

## 与本仓库交付目录的对齐要求

- `docs/prd/`、`docs/tech/`、`docs/qa/`、`docs/release/`：只承载通用产研交付内容。
- `docs/memo/`、`docs/todo/`：可以保留日会与任务闭环机制，不得引入特定业务时段或业务域指标字段。
- `docs/research/`：只保留通用研究活动（技术调研、方案对比、可行性验证），不承载业务域方案研究语义。
